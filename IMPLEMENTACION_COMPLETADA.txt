🎯 CARACTERÍSTICAS DESTACADAS

⭐ TOKENS AUTOMÁTICOS
   └─ Cuando creas un usuario, obtienes token + refreshToken inmediatamente
   └─ No necesitas hacer login adicional
   └─ Ideal para que admin comparta credenciales con nuevo usuario

🔐 PERMISOS GRANULARES
   ├─ users:list, users:view, users:create, users:update
   ├─ users:delete, users:changePassword, users:changeRole
   ├─ users:activate, users:deactivate
   └─ Más de 40 permisos en total por módulo

📋 FILTROS AVANZADOS
   ├─ Paginación (page, limit)
   ├─ Búsqueda (search en nombre y email)
   ├─ Filtro por rol
   ├─ Filtro por estado activo
   └─ Combinables entre sí

🛡️ PROTECCIONES
   ├─ Email único y validado
   ├─ Contraseña encriptada (bcrypt)
   ├─ No elimina último admin
   ├─ No desactiva último admin
   ├─ Usuario inactivo = acceso denegado
   ├─ Roles siempre válidos
   └─ Todas las acciones quedan registradas

🚀 ENDPOINTS - REFERENCIA RÁPIDA

AUTENTICACIÓN
  POST   /api/v1/auth/register           # Registrarse (público)
  POST   /api/v1/auth/login              # Login (público)
  GET    /api/v1/auth/me                 # Perfil actual (privado)
  PUT    /api/v1/auth/me                 # Actualizar perfil (privado)
  PUT    /api/v1/auth/change-password    # Cambiar contraseña (privado)
  POST   /api/v1/auth/logout             # Logout (privado)

USUARIOS
  GET    /api/v1/users                   # Listar (con filtros)
  GET    /api/v1/users/:id               # Obtener uno
  POST   /api/v1/users                   # Crear ⭐ CON TOKENS
  PUT    /api/v1/users/:id               # Actualizar
  PUT    /api/v1/users/:id/password      # Cambiar contraseña
  PUT    /api/v1/users/:id/roles         # Cambiar roles (admin)
  PUT    /api/v1/users/:id/activate      # Activar
  PUT    /api/v1/users/:id/deactivate    # Desactivar
  DELETE /api/v1/users/:id               # Eliminar

INFORMACIÓN
  GET    /api/v1/users/stats/dashboard    # Estadísticas (admin)
  GET    /api/v1/users/roles/list         # Roles disponibles
  GET    /api/v1/users/roles/:rol/perms   # Permisos de rol (admin)

💻 EJEMPLO DE FLUJO COMPLETO

PASO 1: Admin hace login
  POST /api/v1/auth/login
  {"email": "admin@efagram.com", "password": "Admin123!@#"}
  → Recibe: token JWT

PASO 2: Admin accede a modal de crear usuario
  GET /api/v1/users/roles/list
  → Obtiene lista de roles para el SELECT

PASO 3: Admin completa formulario
  - Nombre: "Supervisor Principal"
  - Email: "supervisor@efagram.com"
  - Contraseña: "Supervisor123!@#"
  - Rol: SUPERVISOR

PASO 4: Frontend envía solicitud
  POST /api/v1/users
  Headers: {Authorization: Bearer <token>}
  Body: {nombre, email, password, roles: ["SUPERVISOR"]}

PASO 5: Backend crea usuario y genera tokens
  Response (201):
  {
    "usuario": { ... datos del usuario ... },
    "tokens": {
      "token": "eyJ...",           ← Nuevo usuario puede usar esto
      "refreshToken": "eyJ..."
    }
  }

PASO 6: Frontend muestra éxito
  - Muestra datos del usuario creado
  - Opción de compartir tokens con el nuevo usuario
  - O permitir que inicie sesión directamente con sus credenciales
