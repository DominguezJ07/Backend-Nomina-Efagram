🔑 CARACTERÍSTICAS PRINCIPALES

1️⃣  GENERACIÓN AUTOMÁTICA DE TOKENS
    Cuando creas un usuario, obtienes:
    {
      "tokens": {
        "token": "eyJ...",          // Usar para hacer llamadas
        "refreshToken": "eyJ..."    // Para refrescar sesión
      }
    }

2️⃣  ROLES CON PERMISOS DEFINIDOS
    • ADMIN_SISTEMA - Control total
    • TALENTO_HUMANO - Gestión RRHH
    • SUPERVISOR - Supervisión y reportes
    • JEFE_OPERACIONES - Gestión operativa
    • TRABAJADOR - Ejecución de tareas

3️⃣  PROTECCIONES IMPLEMENTADAS
    ✓ Email único validado
    ✓ Contraseña encriptada (bcrypt)
    ✓ No permite eliminar último admin
    ✓ No permite desactivar último admin
    ✓ Roles siempre válidos
    ✓ Usuario inactivo = acceso denegado
    ✓ Logging de todas las acciones

4️⃣  FILTROS Y BÚSQUEDA AVANZADA
    GET /api/v1/users?page=1&limit=10&search=admin&activo=true&rol=SUPERVISOR
    
    Parámetros:
    - page: Número de página
    - limit: Resultados por página
    - search: Busca en nombre y email
    - activo: Filtro por estado
    - rol: Filtro por rol específico

5️⃣  PERMISOS GRANULARES
    No solo roles, sino permisos específicos:
    - users:list, users:view, users:create, users:update
    - users:delete, users:changePassword, users:changeRole
    - users:activate, users:deactivate
    - Y más para cada módulo...

🎯 FLUJO DE SEGURIDAD IMPLEMENTADO

Request HTTP
    ↓
[1] Middleware authenticate
    ├─ Obtiene token del header
    ├─ Verifica JWT
    ├─ Carga usuario de BD (datos frescos)
    ├─ Valida que esté activo
    └─ Agrega req.user
    ↓
[2] Middleware authorize/checkPermission
    ├─ Valida roles permitidos
    ├─ Comprueba permisos específicos
    └─ Deja pasar o rechaza (403)
    ↓
[3] Controller/Service
    ├─ Ejecuta lógica de negocio
    ├─ Valida datos
    ├─ Modifica BD
    └─ Registra en log
    ↓
Response JSON

✅ VALIDACIONES IMPLEMENTADAS

📧 EMAIL:
   ✓ Formato válido (RFC 5322)
   ✓ Único en BD
   ✓ Normalizado a minúsculas

🔐 CONTRASEÑA:
   ✓ Mínimo 6 caracteres
   ✓ Encriptada con bcrypt (salt: 10)
   ✓ Nunca se retorna en respuestas

👤 NOMBRE:
   ✓ Requerido
   ✓ 2-50 caracteres
   ✓ Sin espacios extra

👑 ROLES:
   ✓ Debe existir en ROLES enum
   ✓ Múltiples roles permitidos
   ✓ Validación de transición de roles

🛡️ ADMIN ÚLTIMO:
   ✓ No se puede eliminar si es único
   ✓ No se puede desactivar si es único
   ✓ No se puede remover del último admin

🚀 CÓMO USAR EN EL FRONTEND

1. CREAR USUARIO DESDE MODAL:
   - Mostrar formulario con campos: nombre, email, contraseña, rol
   - Enviar POST a /api/v1/users con token admin
   - Recibir tokens automáticamente
   - Mostrar confirmación con datos

2. LISTAR USUARIOS:
   - GET a /api/v1/users con filtros opcionales
   - Mostrar paginación
   - Permitir búsqueda por nombre/email
   - Filtrar por rol y estado

3. EDITAR USUARIO:
   - PUT a /api/v1/users/:id
   - Solo campos editables según permisos

4. CAMBIAR ROLES:
   - PUT a /api/v1/users/:id/roles (solo admin)
   - Selectbox con roles disponibles

5. VER PERMISOS:
   - GET a /api/v1/users/roles/:rol/permissions
   - Mostrar matriz de permisos del rol
